How to play

Write a javascript AI to defeat your evil doppelganger in battle. You can choose to either attack or block each turn, where attacking deals damage to your opponent, and blocking restores a little health to yourself. The first to get the others health to zero wins.

settings_ethernet Available properties

player your player class enemy the enemy class debug a class to debug elements with

So it's a game, what's so special about it?

security Scoped javascript

Off course, having users execute code in your application can be dangerous, so I enclosed the scope of the textarea code to make sure the only properties that can be reached are the once I want.

cached Run-time code execution

One of the bigger challenges was to execute user-generated lines of code, especially in combination with the javascript scoping. The textarea code gets wrapped up in a closed function to execute the code and scope it at the same time.

assessment Prepared AI

So how much fun would it be to battle it out with an enemy that doesn't fight back? Not very much. So I had to develop an AI that was both very robust as well as very versatile to account for all possible user-generate code.
